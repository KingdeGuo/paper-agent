# Security Policy / 安全策略

## Supported Versions / 支持的版本

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | ✅ Active support |
| < 1.0   | ❌ Not supported  |

## Reporting a Vulnerability / 报告漏洞

We take security issues seriously. If you discover a security vulnerability in Paper Agent, please follow the steps below:

**Please do NOT report security vulnerabilities via public GitHub Issues.**

### How to report / 如何报告

1. **Email**: Send an email to the maintainers at the address listed in the project's commit history.
2. **GitHub Security Advisory**: Navigate to the repository's **Security** tab and use the "Report a vulnerability" feature.

### What to include / 内容要求

- Type of vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)
- Your contact information

### Response process / 响应流程

1. We will acknowledge your report within **48 hours**.
2. We will investigate and provide an estimated timeline for a fix.
3. Once the fix is ready, we will coordinate a responsible disclosure.
4. After the fix is released, we will publicly acknowledge your contribution (if desired).

## 安全漏洞报告

请**不要**通过公开的 GitHub Issues 报告安全漏洞。

请发送邮件至项目维护者（见提交历史中的邮箱地址），或使用 GitHub 仓库 **Security** 页面的漏洞报告功能。

我们将在 **48 小时内**确认您的报告。
